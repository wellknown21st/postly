# AI Usage Policy and Integration Details

## Integration with LLMs
Postly integrates with both OpenAI (GPT-4o, GPT-4o-mini) and Anthropic (Claude 3.5 Sonnet, Claude 3 Haiku) to dynamically generate content.

### Implementation Strategy
The `aiEngine.js` service is responsible for handling all requests to the LLMs.
1. **Prompt Engineering**: We construct a robust system prompt that instructs the AI to act as a social media manager. It injects specific constraints dynamically based on the requested platforms (e.g., "Must be under 280 characters for Twitter").
2. **Structured Output**: We enforce JSON output formatting from the AI models. This allows the system to easily map generated content to the correct platform records in our database.
3. **Key Management**: The system can fall back to environment-level keys, but prioritizing user-provided keys via the database (encrypted using AES-256).

### Security
We never pass raw user secrets in client-facing requests. All LLM calls are server-side.

## Developer AI Usage Disclosure
During the development of this backend:
- AI assistants were utilized to generate boilerplate code for Express middleware, setup scripts, and standard CRUD controllers.
- Code generated by AI was thoroughly reviewed, modified, and validated to ensure it adheres to the requirements, security practices, and application logic.
- Complex architectural decisions (like Queue backoff strategies and Database Schema mapping) were guided by domain knowledge and supplemented by AI tool documentation.
